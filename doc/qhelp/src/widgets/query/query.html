Query Interface

    The Query Interface is used to create user restrictions on a data set. You can either create restrictions by attribute or a spatial restriction. The operators may vary depending on the selected data type.

To create a restriction, the user must follow this steps:

  1. Select the type of restriction
    • Attribute,
    • Spatial.
  2. Define a restriction.
    • (+) Add a new restriction.
  3. Down below is presented all restrictions defined. This component has two elements:
    • Table: List of restrictions
    • SQL: Restriction defined as SQL (only for visualization purposes of the restriction generated).





For restrictions by attributes, the fields to be defined are:
  • Property: Attribute to be restricted.
  • Operator: List of operators supported for the type of selected data.
  • Value
    • Property: Uses another attribute of the dataset to build the restriction.
    • Value: Uses a specific value to create the restriction.

For spatial restrictions, the fields to be defined are:
  • Geometry Attribute: Attribute with the geometric representation of the selected data set.
  • Operator: List of operators supported for the type of selected data.
  • Layer: Layer to be confronted with the restriction defined.
  • Use Selected Objects: Option used if the user wants to use the selected dataset geometries to build the restriction.
