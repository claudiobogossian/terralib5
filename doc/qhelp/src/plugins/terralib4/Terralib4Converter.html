TerraLib 4.x Converter

The TerraLib 4.x Converteris a tool used to convert data from TerraLib 4.2.2 to TerraLib 5 model. Thus, it  supports only TerraLib databases generated by TerraLib 4.2.2. If the selected database is from a version before 4.2.2, a warning message will appear and this tool must be closed.

It is accessible through:

    Tools > TerraLib4Converter... 

This tool follows a wizard standard in 7 steps to assist the conversion operation configurations:

  1. Connect to a TerraLib4 Database (it is the same interface used to connect with a database in TerraLib4.x).
                 
    1. The operation will be always  Connect.
    2. Select the Database Type  (Access or SQL Server)
    3. Inform the connection parameters (Host, Port, User, Password) if the database type is not Access)
    4. Select the Database... on version 4.2.2
    5. Press Next
      1. If the warning message "The database must be converted to the model 4.2.2"  occur -  press OK
      2. Press Cancel and Exit TerraView.  
      3. Open this database under TerraView 4.2.2 to convert it to this version and them come back here.
  1. Layer Selection - Select the TerraLib 4.x Layer that will be converted to TerraLib 5 and added to the project
    1. Under Layers List  it is possible to Select All or Deselect All
    2. Select All or some of them by checking the box.
    3. Press Next.
  2. Target Data Source -  Select a data source to store the converted data  (Data Source Selector)
    1. Select one of the Available Data Sources  from the list (or create a new one)
    2. Press Next.
  3. Target Raster Folder - Select a folder to Store the raster layers as TIFF files (this step is needed if there is at least one raster layer).
    1. Click on (folder-icon) to define where so store the raster layers as TIFF files.
    2. Press Next.
  4. Resolve/No Name Conflicts - Some layer names might clash with target data source layer names. For those marked with   it is obliged to give another name. Optionally change the names of other layers with the same name as you wish.
    1. Double-Click over the Target Name cell and enter the new name.
    2. Press Commit
  5. Layer Creation - Select the TerraLib4.x Layer converted that you want to create a TerraLib 5 Project Layer.
    1. Under Layers List  it is possible to Select All or Deselect All
    2. Select All or some of them by checking the box.
    3. Press Next.
  6. Theme Creation - Select the Terralib4.x Theme of converted Layers that you want to create a TerraLib 5 Project Layer, with informations like Visual and Grouping.
    1. Under TerraLib 4.x Theme List  it is possible to Select All or Deselect All 
    2. Select All or some of them by checking the box.
    3. Press Finish.
Hints: In TerraLib4.x exist the concept of Layer, View and Theme. The same layer can be represented in different views as themes, and each view holds a specific Spatial Reference System (projection).  By default all of them are selected, but it might spend too much time, and in some cases it is not necessary to convert all of them.

In the example below, the same layer "IBGE_Mun_55mu2500g" would generate three conversions, one for each view and one for the original layer selected at the step 6.  Most of the time it is not necessary as you can do that in the new version. Usually the theme name is used to name the resulting layer avoiding the conflict with the original layer.
